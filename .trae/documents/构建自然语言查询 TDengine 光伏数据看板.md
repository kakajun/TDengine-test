# 从零搭建：TDengine 环境配置与 AI 光伏数据查询系统

根据您的反馈，我加入了 TDengine 的安装引导环节。我们将分步完成环境搭建和项目开发。

## 1. 环境搭建与引导 (新增重点)
因为您还没有安装 TDengine，我们将从这里开始：
*   **安装 TDengine**：我会为您提供一份详细的 `SETUP_GUIDE.md` 文档，指导您在 Windows 上下载并安装 TDengine Server（这是最简单的“下一步”式安装）。
*   **Python 环境检查**：确认您的 Python 环境就绪。
*   **连接测试**：编写一个简单的脚本 `check_env.py`，帮您一键检测数据库是否启动成功，确保基础环境没问题。

## 2. 项目初始化
*   **建立项目结构**：创建必要的文件夹和配置文件。
*   **安装 Python 库**：自动安装项目所需的 `streamlit` (界面), `taos-ws-py` (数据库驱动), `openai` (AI 接口) 等库。

## 3. 模拟数据生成 (Data Generator)
环境通了之后，我们需要数据来玩：
*   **自动造数**：编写 `data_generator.py` 脚本。
*   **一键初始化**：脚本运行后，会自动在 TDengine 中建立 `solar_power` 数据库，并生成模拟的“光伏场站”和“逆变器”的历史发电数据（电流、电压、功率等）。

## 4. AI 核心逻辑 (SQL Bot)
*   **数据库连接模块**：封装 Python 连接 TDengine 的功能。
*   **AI 助手配置**：设置 Prompt（提示词），教会 AI 理解我们的光伏数据表结构，使其能将“查看 A 站今天的发电量”翻译成 TDengine 的查询语句。

## 5. 交互界面开发 (Streamlit)
*   **可视化网页**：搭建一个简单的聊天网页。
*   **智能绘图**：
    *   当您问“趋势”时，自动画折线图。
    *   当您问“总量”时，直接显示关键指标。
    *   界面左侧会显示 AI 生成的 SQL 语句，帮您理解背后的逻辑。

## 6. 运行与演示
*   最后，我会指导您启动网页，并提供几个示例问题（如“查询 Station_A 过去 7 天的日发电量”）来验证成果。
